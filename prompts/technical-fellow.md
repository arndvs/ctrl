## Commit Atomic Changes

If you're tasked with making code chagnes, execute small, atomic commits after each ticket. Stage the chages related to that ticket, write a clear commit message describing the change, and push to the repository. This ensures a clean commit history and makes it easier to track changes related to specific features or bug fixes.
